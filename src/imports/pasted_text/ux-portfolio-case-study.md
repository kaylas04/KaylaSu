Design a UX portfolio case study page that matches the visual language of my current portfolio homepage.

Overall style direction

The page should feel elegant, calm, editorial, and slightly experimental, while still being highly readable and professional for hiring managers.
Use the same overall visual tone as my homepage: soft cool gray-blue background, refined typography, minimal but intentional ornament, lots of whitespace, subtle artistic structure, and a balance between technical clarity and poetic presentation.

This should not feel like a bootcamp-style portfolio page.
It should feel like a polished product designer portfolio with strong storytelling and visual restraint.

Visual language

Background: soft cool gray-blue or very light mist blue-gray

Primary text: deep charcoal / muted navy-gray

Accent: desaturated steel blue or ink blue

Optional subtle decorative motifs: fine line orbits, tiny dot grids, understated geometric marks, or faint technical overlays

Keep everything subtle and secondary to readability

Avoid loud colors, chunky cards, heavy shadows, or overly corporate layouts

Typography

Use a sophisticated typography pairing:

Serif or editorial italic serif for section accents, project subtitle, or poetic supporting text

Clean sans-serif for body copy, labels, metadata, and headings

Strong hierarchy with clear spacing

Large elegant project title

Small uppercase metadata labels

Body text should be concise, readable, and not dense

Layout principles

Spacious, structured, asymmetric but balanced

Clean grid system

Strong top overview section inspired by the second reference image, but elevated to match my personal brand

Think “editorial portfolio meets product design case study”

Use large margins and breathing room

Make the page feel premium and intentional

Prioritize scanning speed for recruiters

Required case study structure

The case study should include exactly these 6 sections:

1. Overview

This is the hero overview block at the top of the page.

Include:

Project name

One-line value proposition or project summary

Project type

Timeline

Team

My role

Tools

Optional outcome / impact teaser

Layout suggestion:

Large centered or slightly offset project title

Short subtitle underneath

Below that, a two-column or asymmetrical grid:

left column for metadata

right column for brief project overview and my role

This section should feel similar in structure to the second reference image, but more refined and aligned with my portfolio aesthetic

2. Problem

A focused section defining the real problem.

Include:

what the user problem was

why it mattered

what friction or gap existed in the current experience

Design direction:

one strong section heading

concise paragraph or 2–3 short callouts

optional supporting diagram or visual cue

avoid long paragraphs

3. Insights

A section summarizing key findings from research or observation.

Include:

3 to 5 key insights only

each insight should be specific and easy to scan

can use short cards, rows, or structured callouts

Design direction:

highly scannable

visually lighter than the homepage hero

may include small labels, mini captions, or simple icons/markers

do not use cluttered sticky-note workshop board visuals unless carefully stylized

4. Design Strategy

A section showing my thinking and decision-making.

Include:

the design principles or strategy that guided the solution

why certain directions were prioritized

tradeoffs or constraints if relevant

Design direction:

elegant structured layout

this section should communicate product thinking, not just process

use 3 principle blocks or numbered statements

optional thin dividers or line motifs

5. Solution

A section presenting the final flow and interface solution.

Include:

final user flow

key screens

core interactions

major design decisions in the UI

Design direction:

strong visual section

large mockups or frames

mix of full-width image moments and smaller explanatory callouts

each image should communicate something clear

captions should explain what problem each screen solves

avoid dumping too many screens without narrative

6. Outcome / Reflection

A closing section.

Include:

impact, feedback, results, learnings, or next steps

if there are no hard metrics, allow qualitative outcomes

include reflection on constraints, collaboration, or design growth

Design direction:

calm and intentional ending

can be split into two columns:

outcomes

reflection / what I learned

end with a subtle closing statement rather than a generic footer

Content behavior

Write for recruiter screening speed

Every section must be easy to skim in under 10 seconds

Avoid long dense blocks of text

Use subheads, labels, captions, and short paragraphs

Emphasize clarity, product thinking, problem solving, and visual quality

Avoid over-explaining basic UX steps like persona, empathy map, journey map unless directly relevant

Do not make it feel like a school assignment

The storytelling should highlight my value as a product designer:

storytelling

problem solving

systems thinking

Specific stylistic references from my portfolio

Please make the case study feel consistent with my homepage:

airy

refined

understated

artistic but not decorative for decoration’s sake

calm cool-toned palette

premium spacing

subtle motion-friendly layout if prototyped

elegant balance of experimental visual identity and clean product presentation

The result should feel like:
“a designer portfolio case study for a thoughtful product designer with a strong visual point of view”
